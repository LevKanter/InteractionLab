Week 6

Quiz Next Week

Next week, there will be a quiz in class. I will give you an image of a simple web page, and you have to reproduce it — as close as you can — using HTML and CSS in a text editor. I'll also provide you with a base HTML file and a base CSS file that has .clearfix defined in it, just in case you need to use it. Any text and images you’ll need will be provided as well (so you can just copy and paste paragraphs of text into your HTML, etc). You will have an hour and a half to complete this.

The quiz will cover HTML and CSS from the first five weeks of the semester. Make sure you understand the following:

  • Working with various header elements, paragraphs, lists, <em> tags, and hyperlinks (the <a> tag).
  • Including an image in the HTML content using the <img> tag.
  • Margins, padding, and borders.Centering a container (such as a <div>) on the page using "auto" margins.
  • Making two block-elements sit side by side using the float property.
  • .clearfix
  • Applying a tiled background image to an element using the background-image and background-repeat property.

To submit your quiz in class, you will have to upload the files to your webspace and put a link to it on your index site (the same site where you've been adding links to your individual homeworks).

There might also be an extra-credit component that involves absolute positioning. Please email me if you have any questions, need help, etc!

Map Project (due 10/24 — not next week)

Please design and build a visualizaton that consists of overlaying various content above specific points on a map. This stems from the maps+flags exercise we did in class, except what you're going to make should be way more fleshed-out and awesome than just putting flags on top of countries. You must have a minimum of three different points on the map where content is overlayed (though I encourage you to have more).

As long as you satisify the essential requirements I'm specifying for this assignment, the content and map that you develop is entirely up to you. You can use a map of the world, a country, a city, an architectural blueprint, a subway map, any kind of map you come up with. The content you choose to associate with your map is up to you as well. You can overlay images, text, video (we'll learn that in class next week!), a combination of different things, etc. It can be serious, funny, informative, whacky, anything! Make something that's genuinely interesting to you. If you feel stuck, let me know and I'll try to help brainstorm ideas. Your project doesn't need to be crazy complex or fancy. It can be simple, but it should be compelling, aesthetically pleasing, and well-executed.

The page you make should be an overall cohesive interface. If somebody not in this class were to land on it, they should be able to understand what it is (i.e. what your map and content are showing). So think about other elements besides just the map itself, such as a title, overview text, etc. Everything should be on one HTML page.

You are responsible for gathering and preparing the images or any other assets that you need. I recommend searching the web for high quality images that are in the public domain (i.e. not copyrighted). For maps, large vector (.eps) files can be particularly useful because you can open them in Illustrator or Photoshop and scale and crop them however you like. Do a search for 'public domain maps .eps' and you'll see how easy it is to find stuff. Ultimately, all media assets should be local in your project. This means, if you find an image online that you want to use, you have to download it and pull it into your HTML/CSS using relative paths, rather than linking directly to its original URL!

Use relative/absolute positioning to layer things above your map. Remember to pay attention to what happens to the positioning of your elements when the size of the browser window changes, as you may not want the layout of your overlays to shift around.

Week 5

Assignment

This week, please begin by devising your own CSS grid system, in a similar vein as what we did in class (we did two different versions. It may be easiest to start by figuring out how many columns you want to fit, and how wide you want your outermost container to be. The dimensions of your grid are up to you, but they should be different than the particular dimensions we used in class. The number of columns is also up to you, but please accommodate at least 8 columns (grid-based layouts in websites tend to consist of somewhere between 8 and 16 columns — we had 10 columns for both grids we did in class). Provide classes in your CSS to define the width for each multiple of columns, like we did with ".x1", ".x2", ".x3", and so on. (While of course these classes can be named however you want — I used x1, x2, etc just to indicate that x1 represented the width of one column, x2 represented two-columns wide, etc etc). Feel free to use that same naming scheme in your grid system, but you're not required to if you want to come up with your own. Also think about how you might want to space your grid columns apart. In our first grid example we didn't have any margins and padding on our ".column" containers, but we created space by adding padding around the contents inside individual columns, whereas in the second variation we actually used margins on the columns themselves.

Then, choose at least 5 images (these can be images of your work (i.e. your portfolio), a collection of photos, drawings, anything you want) and create a webpage that presents them in a grid-based layout, using your CSS system to group images into various columns.

In addition to at least 5 images arranged into some column-based layout, your page should have a title header (i.e. "My Image Gallery" or whatever). Ultimately, your gallery shouldn't show the grid columns like in my example -- those are just there to illustrate how things are alligned with columns. However, as a bonus (not required), actually provide a way to see your grid columns in the background, by adding a class to the <body> like we did with the WORK-IN-PROGRESS class.

Week 4

Assignment

Please take a look at the wireframe version of The Onion we started to produce in class.

In the same vein, I'd like you to pick a different large site with a fixed-width, multi-column layout on its home page, and create your own, lo-res wireframe-esque replica of it. You can choose any site other than The Onion that fits that criteria, but some potential candidates are: The Boston Globe, Rhizome.org, The New York Times.

Don't worry about interactive components, or even graphics and stylistic details. This is all just about breaking the page down into its core containers: the header, the footer, the various content columns, primary nav, secondary nav, and so on. Use dummy content (“lorem ipsum…”, placeholder images) to fill out the various containers. As much as you can, try to eliminate redundancy in your CSS by reusing classes for repetetive layout structures.

Note that my The Onion example is not very far along at all, it is just showing some starting steps. Your wireframed site should still be lo-res, but take it further than I did with the Onion. Try to capture all the essential elements of the page.

As a bonus: please take a look through Grids are Good, a presentation by Khoi Vinh and Mark Boulton about grid-based layouts and their role on the web!

Week 3

Assignment

This week, please begin by finding a short story or some sort of article of text online. It can be anything you want, with the exception of the F. Scott Fitzgerald story I used in class. Some places to look are this American literature site or The New Yorker. You don't have to use a story that is crazy long or anything, but it must bve at least 10 paragraphs (if the story is longer, you can just use the first 10 paragraphs of it).

Once you have a short story, create a basic HTML version of it. Note that I want you to write the HTML yourself, so if you're taking a story from a website don't merely view the page source and copy the markup that's already there. Writing this much HTML is going to be a somewhat tedious exercise (this is actually probably the most you're going to have to write in one swoop all semester). Keep your markup as simple as possible, but pay attention to details such as text that should be emphasized, or text that should be in quotes.

In addition to the actual content of each story, please provide headings for the story's title and the story's author.

Also create a stylesheet for your story. Focus on styling the typography and controllng how things are spaced out (with margins, padding, etc). The only concrete requirements for you CSS is that you use a "web font" at least once — either with @font-face directly, or via a service such as Google Web Fonts (remember back to Week 2) — and you use a background-image at least once.

Course Description

Core Lab: Interaction is about creating user interfaces, and understanding how they work. The main focus will be on making things for the web. We will learn how to write HTML, CSS, and Javascript in order to produce usable and engaging interfaces. To that end, we will be exploring typography, multi-column layouts, navigation menus, image galleries, and more!

This course is designed to be complementary to Core Studio: Interaction. The emphasis here will be placed on learning the technical things you’ll need in order to explore the concepts taught in your studio class.

If you have never written or seen HTML, fear not! We will concentrate on the basics. Everything is going to be okay! If it happens that you already do have experience with these technologies, fear not, you won’t bored– we will hone your craft. At the end of the day, my goal here is for you to be able to produce work in the interactive realm that you are proud of.

Student Expectations

You will be getting homework every week that is usually due by the following class. There will be two larger assignments for which you will have more time (2–3 weeks).

There will be an in-class midterm quiz. Details about this will come in advance.

You are required to keep a website for this class, where you will post links to your completed homework assignments. We will discuss the details of this in the first week.

We will practice and work on things a good amount during class. You are welcome to bring a laptop to work on, but note that it will be fine to use the computers in the classroom. Details about the software tools we’ll actually be using will be discussed in depth, but everything you’ll need is completely free and available in computer labs on campus.

There is a lot of software out there, such as Adobe Dreamweaver, that enables one to produce websites through a visual interface, rather than by writing any code. In order to gain an understanding of how things really work under the hood, you will be required to write the code using a text editor.

The material we are covering in this course can be tough to learn. It’s challenging to write code that doesn’t have errors, and one small error can cause an entire project to break. It takes practice and patience to get into a groove with this stuff, but if you stick to it I think you’ll find that you can make a lot of cool stuff with code that doesn’t have to be extremely complicated.

Academic Integrity

Plagiarism and cheating is bad.

The assignments given in this class will generally be exercises that involve writing HTML, CSS, and Javascript, and for these technologies it’s easy to look up solutions on the internet, and copy somebody else’s code wholesale. You won’t learn anything by doing this.

That said, we may be leveraging open source frameworks and libraries to build things in this class. It’s totally okay to incorporate example code that you find (or that I provide) in your work. But if you do this, have a basic understanding of why you are using a certain piece of code, and always give credit where it’s due.
